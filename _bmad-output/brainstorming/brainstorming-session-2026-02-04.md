# Brainstorming Session Results

**Facilitator:** Ubuntu + Claude
**Date:** 2026-02-04

## Session Overview

**Topic:** PML Repositionnement - Du Workflow à l'Application Conversationnelle

**Goals:**
- Reformuler le Hero de la landing page
- Trouver le bon positionnement marketing
- Garder les 3 piliers (Observable, Agnostique, Collectif) mais changer le message principal

### Context

Suite au brainstorming du 2026-01-28 qui avait défini les 3 piliers (Observable × Agnostique × Collectif), l'émergence de :
- **MCP Apps** - UI dans les chats
- **OpenClaw / Claude Bot** - IA qui agit dans le monde réel
- **Vues composites PML** - Applications complètes hostées

...a révélé un shift de paradigme :

> **Avant:** "Donner à l'IA la possibilité de faire des **actions**"
> **Maintenant:** "Donner à l'IA la possibilité de faire des **applications**"

### Le Vrai Insight

Ce n'est pas juste "workflows + UI". C'est un **nouveau type de navigateur/web** :
- L'humain entre du texte pour communiquer
- Les éléments dont il a besoin s'affichent (UI dynamiques)
- Il interagit avec des boutons/formulaires
- L'application **émerge** de la conversation

**Le chat devient l'application.**

## Phase 1 : First Principles Thinking

### Insights Extraits

**[#1] Le Vrai Problème**
PML ne résout pas "comment orchestrer des MCP". Il résout "comment faire des actions réelles dans le monde depuis une conversation avec l'IA".

**[#2] Atomicité Compositionnelle**
Architecture LEGO : petits workflows réutilisables qui composent des grands workflows. Différent de n8n (monolithique).

**[#3] Triple Simultanéité**
UI + Workflow + Conversation émergent ensemble, dans le même flux. Pas "design → run → interact" mais tout en même temps.

**[#4] Hybride Catalogue/Génération**
Deux modes : suggérer une capacité existante OU générer dynamiquement via code. L'IA choisit le plus efficace.

**[#5] Réponse Instinctive**
"Ça permet de composer des interfaces utilisateur dans le chat" - la vérité la plus simple.

**[#6] Problème de Vocabulaire**
L'ambition est agnostique mais le jargon ("modèle", "LLM", "runtime") est incompréhensible pour le grand public.

**[#7] La Formulation Gagnante**
"Interfaces utilisateur à la demande" - 5 mots qui capturent tout sans jargon.

### Hero Émergent (Draft)

> **PML**
> Des interfaces à la demande.
> _Parlez. L'application apparaît._

## Phase 2 : Alien Anthropologist

### Insights Extraits

**[#1] L'Invocation**
"Invoquer des outils par la parole" - pas créer, pas développer. Invoquer. Comme de la magie.

**[#2] L'Intuition Collective**
SHGAT + GRU = intuition apprise. Le système "sent" ce que tu veux faire grâce à l'apprentissage collectif.

**[#3] Deux Niveaux**
PML (infra/runtime) vs Application (ce que les gens utilisent). Comme électricité vs appareils électriques.

**[#4] Écosystème Dual**
Builders (code) + Users (intention) - mais frontière poreuse.

**[#5] Le Continuum Créatif**
Pas de frontière nette créer/utiliser. Chaque adaptation par un agent crée automatiquement une nouvelle capacité. Organisme qui grandit.

**[#6] Le Piège de la Fascination Technique**
"Vivant/évolutif" fascine les créateurs mais ne vend pas. Les utilisateurs veulent : qu'est-ce que j'obtiens ?

**[#7] La Valeur Concrète**
- Builders : Composer des dashboards type Grafana dans le chat
- Users : UN workflow, UNE interface au lieu de 10 apps
- Entreprises : Power BI-style à la volée, sans setup, moins cher

### Inquiétude Identifiée

> "Power BI va finir par le faire aussi"

→ Comment se positionner quand le géant peut copier ?

## Phase 3 : Cross-Pollination

### Patterns Analysés

| Pattern | Application PML |
|---------|----------------|
| **Notion vs Microsoft** | "On n'est pas meilleur, on est unifié" |
| **Figma vs Adobe** | "On change OÙ et COMMENT ça se passe" |
| **Slack vs Teams** | "On n'est pas pour tout le monde, on est pour ceux qui veulent le meilleur" |
| **Android vs iPhone** | "Le standard ouvert, pas le jardin fermé" |
| **WordPress vs Wix** | "Communauté > Corporation" |

### Insight Clé : La Transparence Composable

| ChatGPT/OpenAI | PML |
|----------------|-----|
| Modèle = boîte noire | Workflows = explicites |
| Opaque | Observable |
| Vendor lock-in | Agnostique |

**OpenAI ne peut PAS copier l'aspect observable sans casser leur business model.**

### Segment Identifié

**Entreprises** = besoin d'auditer, contrôler, compliance → PML gagne vs ChatGPT

## Exploration Approfondie : La Vraie Vision

### Le Problème des Slogans Réducteurs

Tentatives abandonnées car trop réductrices :
- "Interfaces à la demande" → ressemble à Lovable/Bolt
- "Vos outils dans le chat" → ne capture pas l'orchestration
- "Automatisation DevOps" → trop niche

### Ce Qui Fait Vraiment Vibrer

1. **Composabilité** - assembler des trucs ensemble
2. **Observabilité** - voir ce qui se passe
3. **Communautaire** - pas une entreprise, une communauté qui crée
4. **Accessible via le chat** - peu importe le "navigateur"

| Navigateur | Chat (Claude, ChatGPT...) |
| HTTP | MCP |
| **Gateway/CDN** | **PML** |
