# Protocol Mapping

## MCP Apps JSON-RPC <-> Platform Mapping

### Initialization Handshake

#### MCP Apps Standard Flow
```
App -> Host:  ui/initialize { capabilities, clientInfo, protocolVersion }
Host -> App:  { protocolVersion, hostCapabilities, hostInfo, hostContext }
App -> Host:  ui/notifications/initialized {}
```

#### Telegram Bridge Flow
```
1. Telegram.WebApp SDK auto-initializes (global available)
2. Bridge reads platform state:
   - Telegram.WebApp.colorScheme -> hostContext.theme
   - Telegram.WebApp.viewportHeight -> hostContext.containerDimensions
   - Telegram.WebApp.platform -> hostContext.platform
   - Telegram.WebApp.initDataUnsafe -> auth context
3. Bridge connects WebSocket to resource server
4. Resource server synthesizes ui/initialize response with platform data
5. Bridge dispatches response to App class
6. App class sends ui/notifications/initialized
7. Resource server sends tool-input/tool-result as appropriate
```

#### LINE LIFF Bridge Flow
```
1. liff.init({ liffId }) -> await
2. Bridge reads platform state:
   - liff.getOS() -> hostContext.platform
   - liff.getLanguage() -> hostContext.locale
   - liff.isInClient() -> determine capabilities
3. Same WebSocket flow as Telegram
```

### Tool Calls (App -> Server)

| MCP JSON-RPC | Direction | Bridge Action |
|-------------|-----------|---------------|
| `tools/call` | App -> Host | Forward via WS to resource server, which calls MCP server |
| `resources/read` | App -> Host | Forward via WS to resource server, which calls MCP server |

These pass through unchanged. The bridge is transparent for server-bound requests.

### Host Notifications (Server -> App)

| MCP JSON-RPC | Direction | Bridge Action |
|-------------|-----------|---------------|
| `ui/notifications/tool-input` | Host -> App | Resource server sends via WS, bridge dispatches as MessageEvent |
| `ui/notifications/tool-input-partial` | Host -> App | Same (streaming support) |
| `ui/notifications/tool-result` | Host -> App | Same |
| `ui/notifications/tool-cancelled` | Host -> App | Same |
| `ui/notifications/host-context-changed` | Host -> App | Generated by bridge when platform state changes |
| `ui/resource-teardown` | Host -> App | Generated when webview closes |

### UI Requests (App -> Host)

| MCP JSON-RPC | Direction | Platform Mapping |
|-------------|-----------|-----------------|
| `ui/open-link` | App -> Host | `Telegram.WebApp.openLink(url)` / `liff.openWindow({ url })` |
| `ui/message` | App -> Host | `Telegram.WebApp.sendData(text)` (caution: closes app) / `liff.sendMessages()` |
| `ui/update-model-context` | App -> Host | Stored in resource server session (no LLM in messaging platforms) |
| `ui/request-display-mode` | App -> Host | `Telegram.WebApp.expand()` / `Telegram.WebApp.requestFullscreen()` |

### Platform Events -> MCP Notifications

#### Telegram Events

| Telegram Event | MCP Notification |
|---------------|-----------------|
| `themeChanged` | `ui/notifications/host-context-changed { theme }` |
| `viewportChanged` | `ui/notifications/host-context-changed { containerDimensions }` |
| `activated` | (no MCP equivalent, internal use) |
| `deactivated` | (trigger session cleanup) |
| `mainButtonClicked` | Custom notification via resource server |
| `backButtonClicked` | Custom notification via resource server |

#### LINE LIFF Events

| LIFF Event | MCP Notification |
|-----------|-----------------|
| Page visibility change | `ui/notifications/host-context-changed` |
| LIFF close | `ui/resource-teardown` |

## Host Context Mapping

### Telegram -> HostContext

```typescript
function telegramToHostContext(): HostContext {
  const tg = window.Telegram.WebApp;
  return {
    theme: tg.colorScheme,                        // 'light' | 'dark'
    styles: {
      variables: {
        '--color-background-primary': tg.themeParams.bg_color,
        '--color-background-secondary': tg.themeParams.secondary_bg_color,
        '--color-text-primary': tg.themeParams.text_color,
        '--color-text-secondary': tg.themeParams.subtitle_text_color,
        '--color-border-primary': tg.themeParams.section_separator_color,
        '--color-ring-primary': tg.themeParams.accent_text_color,
      },
    },
    containerDimensions: {
      width: window.innerWidth,
      maxHeight: tg.viewportStableHeight,
    },
    platform: 'mobile',
    safeAreaInsets: {
      top: tg.safeAreaInset?.top ?? 0,
      right: tg.safeAreaInset?.right ?? 0,
      bottom: tg.safeAreaInset?.bottom ?? 0,
      left: tg.safeAreaInset?.left ?? 0,
    },
    locale: navigator.language,
    timeZone: Intl.DateTimeFormat().resolvedOptions().timeZone,
  };
}
```

### LINE LIFF -> HostContext

```typescript
async function liffToHostContext(): Promise<HostContext> {
  return {
    theme: 'light',                               // LIFF doesn't expose theme
    containerDimensions: {
      width: window.innerWidth,
      maxHeight: window.innerHeight,
    },
    platform: liff.getOS() === 'ios' || liff.getOS() === 'android' ? 'mobile' : 'web',
    locale: liff.getLanguage(),
    timeZone: Intl.DateTimeFormat().resolvedOptions().timeZone,
  };
}
```

## Host Capabilities Mapping

```typescript
function platformToHostCapabilities(platform: string): HostCapabilities {
  const base: HostCapabilities = {
    serverTools: { listChanged: false },
    serverResources: { listChanged: false },
    logging: {},
  };

  switch (platform) {
    case 'telegram':
      return {
        ...base,
        openLinks: {},                             // Telegram.WebApp.openLink()
        // No sandbox (we are the sandbox)
      };

    case 'line':
      return {
        ...base,
        openLinks: {},                             // liff.openWindow()
      };

    default:
      return base;
  }
}
```

## Message Translation Example

### tools/call Round Trip

```
1. MCP App calls: app.callServerTool({ name: "get-time", arguments: {} })

2. App class generates:
   {
     "jsonrpc": "2.0",
     "id": 1,
     "method": "tools/call",
     "params": { "name": "get-time", "arguments": {} }
   }

3. App class does: window.parent.postMessage(message, "*")

4. Bridge adapter intercepts, sends via WebSocket:
   -> ws.send(JSON.stringify(message))

5. Resource server receives, forwards to MCP server:
   -> mcpClient.callTool("get-time", {})

6. MCP server returns:
   { content: [{ type: "text", text: "2026-02-09T12:00:00Z" }] }

7. Resource server wraps as JSON-RPC response:
   {
     "jsonrpc": "2.0",
     "id": 1,
     "result": {
       "content": [{ "type": "text", "text": "2026-02-09T12:00:00Z" }]
     }
   }

8. Sends via WebSocket to bridge adapter

9. Bridge adapter dispatches as MessageEvent:
   window.dispatchEvent(new MessageEvent('message', { data: response }))

10. App class receives, fires ontoolresult callback
```
