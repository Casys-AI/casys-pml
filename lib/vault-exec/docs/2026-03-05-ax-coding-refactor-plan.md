# 2026-03-05 — AX-Coding Full Refactor Plan (VaultExec)

## Goal
Refactor `src/` to a machine-first architecture optimized for Agent Experience (AX): strict contracts, deterministic execution, explicit traces, and policy-driven decomposition.

## Six high-level pillars

1. **Contracts-first core**
   - Define/version canonical contracts for intent, action, result, and trace events.
   - Move only truly reusable primitives to `core`.

2. **Trace as source-of-truth**
   - Make event model explicit and immutable.
   - Normalize OpenClaw ingest into canonical trace events with correlation ids.

3. **Policy engine for L1/L2/L3**
   - Isolate decomposition/routing policy in dedicated module(s).
   - L2 production-ready across major tools; L3 optional and conservative.
   - Attach confidence + fallback reason for every decision.

4. **Runtime/action isolation**
   - Keep execution logic pure and atomic.
   - Remove hidden coupling between runtime, ingest, and markdown projection.

5. **Projection layer**
   - Treat markdown/report outputs as projections from trace+policy, not business logic.
   - Add decomposition coverage reporting as first-class projection.

6. **Quality gate + dead-code elimination**
   - Remove stale/legacy paths.
   - Add tests per layer (contracts, policy, ingest, projections).
   - Enforce deterministic behavior + explicit fallback in ambiguous cases.

7. **Event-native architecture (Deno-first, OTel-ready)**
   - Add an internal event envelope (`eventId`, `traceId`, `type`, `ts`, `actor`, `payload`, `schemaVersion`).
   - Use a lightweight internal event bus (Deno-native primitives) for domain/application events.
   - Keep OpenTelemetry as an **optional adapter** (opt-in), not a domain dependency.
   - Support pluggable exporters/adapters (`markdown`, `kv`, `logs`, `otel`) so observability can evolve without refactoring business logic.

## Execution waves

### Wave A — Structural extraction (high priority)
- Carve boundaries: `contracts/`, `trace/`, `policy/`, `runtime/`, `projections/`.
- Move shared types/interfaces to core and remove duplicated types.

### Wave B — L2 completion (MVP target)
- Extend L2 rules to major tools (`process`, `browser`, `web_*`, `message`, `cron`, `sessions_*`, `memory_*`, etc.).
- Improve `exec` normalization (wrappers context + main action family).
- Produce L2 coverage report with hit/fallback rates.

### Wave C — Hardening
- Remove dead code/legacy branches.
- Stabilize tests and CLI entry points.
- Document architecture invariants + rule versioning.
- **Rattrapage legacy**: aligner l’existant aux conventions AX (doc co-localisée, nommage lowercase/kebab-case, suppression docs orphelines, cohérence readme/contract/examples par module).

## Definition of Done (this refactor)
- Clear module boundaries with no cross-layer leakage.
- L2 working across major tools with conservative fallback.
- Coverage report generated by pipeline.
- Tests green and no obvious dead code in touched modules.
- Architecture note updated for future contributors/agents.
