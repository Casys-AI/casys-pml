## Goal

Refactor `src/` to a machine-first architecture optimized for Agent Experience
(AX): strict contracts, deterministic execution, explicit traces, and
policy-driven decomposition.

## Six high-level pillars

1. **Contracts-first core**
   - Define/version canonical contracts for intent, action, result, and trace
     events.
   - Move only truly reusable primitives to `core`.

2. **Trace as source-of-truth**
   - Make event model explicit and immutable.
   - Normalize OpenClaw ingest into canonical trace events with correlation ids.

3. **Policy engine for L1/L2/L3**
   - Isolate decomposition/routing policy in dedicated module(s).
   - L2 production-ready across major tools; L3 optional and conservative.
   - Attach confidence + fallback reason for every decision.

4. **Runtime/action isolation**
   - Keep execution logic pure and atomic.
   - Remove hidden coupling between runtime, ingest, and markdown projection.

5. **Projection layer**
   - Treat markdown/report outputs as projections from trace+policy, not
     business logic.
   - Add decomposition coverage reporting as first-class projection.

6. **Quality gate + dead-code elimination**
   - Remove stale/legacy paths.
   - Add tests per layer (contracts, policy, ingest, projections).
   - Enforce deterministic behavior + explicit fallback in ambiguous cases.

7. **Event-native architecture (Deno-first, OTel-ready)**
   - Add an internal event envelope (`eventId`, `traceId`, `type`, `ts`,
     `actor`, `payload`, `schemaVersion`).
   - Use a lightweight internal event bus (Deno-native primitives) for
     domain/application events.
   - Keep OpenTelemetry as an **optional adapter** (opt-in), not a domain
     dependency.
   - Support pluggable exporters/adapters (`markdown`, `kv`, `logs`, `otel`) so
     observability can evolve without refactoring business logic.

8. **Trace data strategy (args-first, privacy-first)**
   - KV is the operational source of truth for traces (markdown is a projection
     only).
   - Persist full argument payloads (`args_raw`) + normalized payloads
     (`args_norm`) for queryability and future simulation.
   - Replace synthetic-trace dependence progressively with real execution traces
     from OpenClaw ingest.
   - Add anonymization at ingest time (PII redaction/tokenization) to prepare
     centralized/federated learning paths.
   - Keep remote learning as a later phase; current phase optimizes local
     anonymized trace quality.

9. **GRU beam / dry-run exploration**
   - Use Beam search over action sequences, conditioned by argument variants.
   - Simulate alternative paths via dry-run on normalized args without mutating
     production state.
   - Store beam candidates and outcomes as trace artifacts for
     replay/evaluation.
