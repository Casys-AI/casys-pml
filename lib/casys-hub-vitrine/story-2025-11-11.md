## Our Projects

### Active Projects

#### Casys MCP Gateway
**Context Intelligence for AI Agents**

**The Problem:**
- 30-50% of LLM context consumed by MCP schemas before any interaction
- Sequential execution causes 5x latency cascade
- Maximum 7-8 MCP servers before context overflow

**Our Solution:**
- **Context Optimization**: Vector search for on-demand tool loading (30-50% → <5%)
- **DAG Execution**: Intelligent parallelization (5x latency reduction)
- **SQLite-First**: Zero-infrastructure, portable, edge-ready

**Results:**
- Support 15+ MCP servers simultaneously
- 90% context recovered for useful conversations
- Workflows 5x faster

**Tech Stack:** Deno, TypeScript, SQLite, sqlite-vec
**License:** MIT Open Source
**Status:** Epic 3 in progress

[Learn More] [GitHub] [Documentation]

---

#### CasysDB
**Your Data, Naturally Connected**

Embedded graph database with Git-like branches and time travel.

**The Problem:**
- Graph databases require server management (Neo4j, ArangoDB)
- SQL databases aren't optimized for graph queries
- No embedded solution with MVCC + time travel

**Our Solution:**
- **Embedded**: Zero servers, runs in your process (like SQLite)
- **ISO GQL**: Standard graph query language (MATCH, WHERE, RETURN)
- **MVCC + PITR**: Isolated transactions + Point-in-Time Recovery
- **Git-like Branches**: Test changes safely, branch from any commit
- **Python ORM**: Type-safe fluent API (like SQLAlchemy for graphs)

**Use Cases:**
- Knowledge graphs for RAG systems
- Social networks and recommendations
- Network analysis and fraud detection
- Semantic search and ontologies

**Tech Stack:** Rust engine, Python/TypeScript bindings, zero dependencies
**License:** MIT Open Source
**Status:** Active Development

[Documentation](https://y-wheat-rho-70.vercel.app/docs/getting-started/installation/) [GitHub](https://github.com/casysai/casysdb)
