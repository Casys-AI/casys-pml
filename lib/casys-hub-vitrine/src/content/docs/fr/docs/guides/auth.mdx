Ajouter l'auth a un serveur MCP signifie generalement brancher la recuperation JWKS, la validation de tokens, la rotation de cles, la verification des scopes, et les endpoints de metadonnees soi-meme. Avec `@casys/mcp-server`, c'est une seule ligne.

## Presets d'auth

Choisissez votre fournisseur d'identite. Chaque preset gere automatiquement les URLs d'issuer, les endpoints JWKS, et la rotation de cles :

<Tabs>
  <TabItem label="Google">
    ```typescript
    import {
      ConcurrentMCPServer,
      createGoogleAuthProvider,
    } from "@casys/mcp-server";

    const server = new ConcurrentMCPServer({
      name: "my-api",
      version: "1.0.0",
      auth: {
        provider: createGoogleAuthProvider({
          audience: "https://my-mcp.example.com",
          resource: "https://my-mcp.example.com",
        }),
      },
    });
    ```
  </TabItem>

## Verification des scopes

Les outils peuvent declarer des scopes requis. Le framework verifie automatiquement le token avant l'execution du handler :

```typescript
server.registerTool(
  {
    name: "delete_user",
    description: "Delete a user account",
    inputSchema: {
      type: "object",
      properties: { userId: { type: "string" } },
      required: ["userId"],
    },
    requiredScopes: ["admin", "users:delete"],
  },
  async ({ userId }) => {
    // This code only runs if the token has BOTH admin AND users:delete
    await deleteUser(userId);
    return { deleted: true };
  },
);
```

<Aside>
  La verification des scopes est **basee sur le AND** — le token doit posseder *tous* les scopes listes. Besoin d'une logique OR ? Implementez-la dans un [middleware personnalise](/fr/docs/guides/middleware/).
</Aside>

## Avance : JwtAuthProvider

Pour un controle total sur les parametres de validation, utilisez `JwtAuthProvider` directement :

```typescript
import { JwtAuthProvider } from "@casys/mcp-server";

const provider = new JwtAuthProvider({
  issuer: "https://accounts.google.com",
  audience: "https://my-mcp.example.com",
  resource: "https://my-mcp.example.com",
  authorizationServers: ["https://accounts.google.com"],
  jwksUri: "https://www.googleapis.com/oauth2/v3/certs",
  scopesSupported: ["read", "write"],
});

const authInfo = await provider.verifyToken(token);
// { subject, clientId, scopes, claims, expiresAt } — or null if invalid
```

## Decouverte automatique (RFC 9728)

Quand l'auth est configuree sur un serveur HTTP, le framework sert automatiquement un endpoint de decouverte. Les clients MCP l'utilisent pour savoir comment s'authentifier :

```
GET /.well-known/oauth-protected-resource
```

```json
{
  "resource": "https://my-mcp.example.com",
  "authorization_servers": ["https://accounts.google.com"],
  "scopes_supported": ["read", "write"],
  "bearer_methods_supported": ["header"]
}
```

<Aside type="tip">
  Pas besoin de configurer cela manuellement. C'est derive de votre configuration d'auth provider et servi automatiquement.
</Aside>
