## 认证预设

选择你的身份提供商。每个预设自动处理 issuer URL、JWKS 端点和密钥轮换：

<Tabs>
  <TabItem label="Google">
    ```typescript
    import {
      ConcurrentMCPServer,
      createGoogleAuthProvider,
    } from "@casys/mcp-server";

    const server = new ConcurrentMCPServer({
      name: "my-api",
      version: "1.0.0",
      auth: {
        provider: createGoogleAuthProvider({
          audience: "https://my-mcp.example.com",
          resource: "https://my-mcp.example.com",
        }),
      },
    });
    ```
  </TabItem>
  <TabItem label="Auth0">
    ```typescript
    import {
      ConcurrentMCPServer,
      createAuth0AuthProvider,
    } from "@casys/mcp-server";

    const server = new ConcurrentMCPServer({
      name: "my-api",
      version: "1.0.0",

          domain: "my-tenant.auth0.com",
          audience: "https://my-mcp.example.com",
    ```

## Scope 验证

工具可以声明所需的 scope。框架在 handler 执行前自动检查 token：

```typescript
server.registerTool(
  {
    name: "delete_user",
    description: "Delete a user account",
    inputSchema: {
      type: "object",
      properties: { userId: { type: "string" } },
      required: ["userId"],
    },
    requiredScopes: ["admin", "users:delete"],
  },
  async ({ userId }) => {
    // This code only runs if the token has BOTH admin AND users:delete
    await deleteUser(userId);
    return { deleted: true };
  },
);
```

<Aside>
  Scope 检查是**基于 AND 的** — token 必须拥有*所有*列出的 scope。需要 OR 逻辑？在[自定义中间件](/zh/docs/guides/middleware/)中实现。
</Aside>

## 进阶：JwtAuthProvider

要完全控制验证参数，直接使用 `JwtAuthProvider`：

```typescript
import { JwtAuthProvider } from "@casys/mcp-server";

const provider = new JwtAuthProvider({
  issuer: "https://accounts.google.com",
  audience: "https://my-mcp.example.com",
  resource: "https://my-mcp.example.com",
  authorizationServers: ["https://accounts.google.com"],
  jwksUri: "https://www.googleapis.com/oauth2/v3/certs",
  scopesSupported: ["read", "write"],
});

const authInfo = await provider.verifyToken(token);
// { subject, clientId, scopes, claims, expiresAt } — or null if invalid
```

## 自动发现 (RFC 9728)

在 HTTP 服务器上配置认证后，框架自动提供发现端点。MCP 客户端用它来确定如何认证：

```
GET /.well-known/oauth-protected-resource
```

```json
{
  "resource": "https://my-mcp.example.com",
  "authorization_servers": ["https://accounts.google.com"],
  "scopes_supported": ["read", "write"],
  "bearer_methods_supported": ["header"]
}
```

<Aside type="tip">
  无需手动设置。这是从你的 auth provider 配置中派生的，自动提供服务。
</Aside>
