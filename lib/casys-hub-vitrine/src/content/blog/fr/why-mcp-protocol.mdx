## Le problème du chargeur

Vous vous souvenez de 2010 ? Chaque marque de téléphone avait son propre chargeur. Voyager, c'était trimballer un sac de câbles.

Et puis l'USB-C est arrivé. Un seul câble, tous les appareils.

**Les outils IA ont exactement le même problème aujourd'hui.**

Chaque fournisseur d'IA a sa propre façon de connecter les outils :
- OpenAI fait comme ci
- Anthropic fait comme ça
- Google fait encore autrement
- Et ainsi de suite...

Vous développez un outil pour l'un ? Il faut le reconstruire pour les autres.

<ExcalidrawDiagram src="mcp-without" alt="Sans MCP : N adaptateurs" />

## MCP entre en scène

Le **Model Context Protocol** est le USB-C des outils IA.

Un seul standard. Compatible partout.

<ExcalidrawDiagram src="mcp-with" alt="Avec MCP : un seul standard" />

Développez votre outil une fois. Il fonctionne avec Claude, GPT, les modèles locaux, et tout ce qui viendra après.

## Ce que ça change pour vous

### 1. Pas d'enfermement

Vous utilisez Claude aujourd'hui ? Parfait. Envie d'essayer GPT demain ? Vos outils vous suivent.

Vous n'êtes plus marié à un seul fournisseur d'IA.

### 2. Plus d'outils, moins d'effort

L'écosystème MCP grandit vite :
- Gestion de fichiers
- Intégration GitHub
- Accès aux bases de données
- Navigation web
- Et des centaines d'autres...

Ces outils fonctionnent immédiatement. Pas de configuration, pas de casse-tête d'intégration.

### 3. Pérennité garantie

De nouveaux modèles IA apparaissent en permanence. Avec MCP, ils peuvent utiliser vos outils existants dès le premier jour.

Votre investissement dans les outils ne devient jamais obsolète.

## Ce que PML fait avec MCP

PML se connecte aux outils MCP et les rend plus intelligents :

| Sans PML | Avec PML |
| -------- | -------- |
| Vous choisissez quel outil utiliser | PML suggère le bon outil |
| Vous enchaînez les outils manuellement | PML construit les workflows automatiquement |
| Les outils sont isolés | Les outils apprennent à travailler ensemble |

MCP fournit les connexions. PML fournit l'intelligence.

## L'effet écosystème

Voilà ce qui est enthousiasmant : plus il y a d'outils qui parlent MCP, plus l'écosystème gagne en puissance.

- **Les créateurs d'outils** touchent plus d'utilisateurs
- **Les développeurs IA** accèdent à plus de fonctionnalités
- **Les utilisateurs** obtiennent de meilleurs assistants IA

Tout le monde y gagne quand on s'accorde sur le standard.

## En résumé

MCP fait pour les outils IA ce que l'USB-C a fait pour les appareils.

Un seul standard. Compatibilité universelle. Fini le sac de câbles.

PML est construit sur MCP parce que nous croyons aux standards ouverts. Vos outils doivent fonctionner partout, pas seulement chez un fournisseur.

**L'avenir de l'IA est connecté. MCP est la connexion.**
