## Le problème

Chaque jour, vous répétez les mêmes séquences :

- Pull le code, lancer les tests, push les changements
- Vérifier les données, générer un rapport, envoyer un email
- Rechercher un sujet, rédiger un document, obtenir un retour

Vous connaissez ces enchaînements par coeur. Mais vos outils IA ? Ils repartent de zéro à chaque fois.

**Et si votre IA s'en souvenait ?**

## Comment PML apprend

PML observe votre façon de travailler. Pas pour vous espionner — pour vous aider.

Quand vous terminez une tâche, PML note :

- Quels outils vous avez utilisés
- Dans quel ordre
- Si ça a fonctionné

Faites la même chose quelques fois, et PML fait le lien.

## De l'habitude à la capacité

Une **capacité**, c'est un workflow que PML a appris en vous observant.

| Ce que vous avez fait | Ce que PML a appris |
| --------------------- | ------------------- |
| git add → commit → push (5 fois) | Capacité « Git Publish » |
| requête DB → formatage CSV → email (3 fois) | Capacité « Génération de rapport » |
| recherche docs → résumé → sauvegarde (4 fois) | Capacité « Recherche & Archivage » |

Vous n'avez rien programmé. Ces capacités ont émergé de vos habitudes.

<ExcalidrawDiagram src="emerge-tools-vs-capability" alt="Des outils aux capacités" />

## Pourquoi c'est important

### 1. Aucune configuration nécessaire

L'automatisation traditionnelle :
```
"Définir le workflow 'deploy' avec les étapes A, B, C, les permissions X, Y, Z..."
```

Avec PML :
```
Travaillez, tout simplement. Les patterns deviennent des capacités automatiquement.
```

### 2. C'est personnel

Vos collègues ont peut-être des workflows différents. Pas de souci.

PML apprend **vos** habitudes, pas un modèle générique. Mêmes outils, capacités différentes — selon la façon dont **vous** les utilisez.

### 3. Ça évolue

Vous avez changé votre workflow ? Ajouté une étape ? PML s'adapte.

- Les nouveaux patterns sont reconnus
- Les anciens patterns que vous n'utilisez plus s'estompent
- Vos capacités restent en phase avec votre façon réelle de travailler

## Ce que ça change au quotidien

**Avant PML :**
> « Je dois déployer. Voyons... d'abord je lance les tests, puis le build, puis le push vers le registry, puis la mise à jour du service, puis je préviens l'équipe... »

**Avec PML :**
> « Déploie ça. »
>
> PML : « C'est noté. Je lance votre workflow de déploiement habituel. »

La charge mentale disparaît. Vous vous concentrez sur **ce que** vous voulez, pas sur **comment** le faire.

## Combiner les capacités

C'est là que ça devient vraiment puissant : les capacités se composent.

Vous avez :
- La capacité « Lancer les tests »
- La capacité « Build Docker »
- La capacité « Déployer dans le cloud »

Utilisez-les ensemble quelques fois, et PML crée :
- La méta-capacité « Release complète »

<ExcalidrawDiagram src="emerge-meta-capabilities" alt="Méta-capacités" />

Vos workflows se composent. La complexité est absorbée. Vous travaillez à un niveau d'abstraction supérieur.

## En résumé

PML ne vous demande pas de définir vos workflows à l'avance.

Il observe. Il apprend. Il suggère.

Plus vous l'utilisez, mieux il vous connaît. Mieux il vous connaît, moins vous vous répétez.

**Votre IA se souvient enfin de comment vous travaillez.**
