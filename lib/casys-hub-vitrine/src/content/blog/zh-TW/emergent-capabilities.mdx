## 问题所在

每天，你都在重复同样的操作序列：

- 拉取代码、运行测试、推送更改
- 检查数据、生成报告、发送邮件
- 调研主题、起草文档、收集反馈

这些模式你早已烂熟于心。但你的 AI 工具呢？每次都从零开始。

**如果你的 AI 能记住这些呢？**

## PML 如何学习

PML 观察你的工作方式——不是为了监控，而是为了帮你。

当你完成一项任务时，PML 会注意到：

- 你使用了哪些工具
- 使用的顺序是什么
- 最终结果是否成功

重复几次同样的操作，PML 就能把这些点连起来。

## 从模式到能力

**能力**（Capability）是 PML 通过观察你的行为而学到的工作流。

| 你做了什么 | PML 学到了什么 |
| ------------ | ---------------- |
| git add → commit → push（5 次） | "Git 发布"能力 |
| 查询数据库 → 格式化 CSV → 发送邮件（3 次） | "报告生成"能力 |
| 搜索文档 → 摘要总结 → 保存归档（4 次） | "调研与归档"能力 |

你没有编写过这些流程，它们从你的习惯中**自然涌现**。

## 为什么这很重要

### 1. 无需任何配置

传统自动化方式：
```
"定义工作流'部署'，包含步骤 A、B、C，权限 X、Y、Z……"
```

使用 PML：
```
正常工作就好。模式会自动成为能力。
```

### 2. 因人而异

你的同事可能有不同的工作流，这完全没问题。

PML 学习的是**你的**模式，而不是某个通用模板。同一个工具，不同的能力——取决于**你**怎么用它。

### 3. 持续进化

改变了工作流？增加了新步骤？PML 会自动适应。

- 新的模式会被识别
- 你不再使用的旧模式会逐渐淡化
- 你的能力始终与你实际的工作方式保持同步
