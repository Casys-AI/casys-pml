## 問題所在

每天，你都在重複同樣的操作序列：

- 拉取程式碼、執行測試、推送更改
- 檢查資料、產生報告、發送郵件
- 調研主題、起草文件、收集回饋

這些模式你早已爛熟於心。但你的 AI 工具呢？每次都從零開始。

**如果你的 AI 能記住這些呢？**

## PML 如何學習

PML 觀察你的工作方式——不是為了監控，而是為了幫你。

當你完成一項任務時，PML 會注意到：

- 你使用了哪些工具
- 使用的順序是什麼
- 最終結果是否成功

重複幾次同樣的操作，PML 就能把這些點連起來。

## 從模式到能力

**能力**（Capability）是 PML 透過觀察你的行為而學到的工作流。

| 你做了什麼 | PML 學到了什麼 |
| ------------ | ---------------- |
| git add → commit → push（5 次） | 「Git 發佈」能力 |
| 查詢資料庫 → 格式化 CSV → 發送郵件（3 次） | 「報告產生」能力 |
| 搜尋文件 → 摘要總結 → 儲存歸檔（4 次） | 「調研與歸檔」能力 |

你沒有編寫過這些流程，它們從你的習慣中**自然湧現**。

<ExcalidrawDiagram src="emerge-tools-vs-capability" alt="從工具到能力" />

## 為什麼這很重要

### 1. 無需任何配置

傳統自動化方式：
```
"定義工作流'部署'，包含步驟 A、B、C，權限 X、Y、Z……"
```

使用 PML：
```
正常工作就好。模式會自動成為能力。
```

### 2. 因人而異

你的同事可能有不同的工作流，這完全沒問題。

PML 學習的是**你的**模式，而不是某個通用模板。同一個工具，不同的能力——取決於**你**怎麼用它。

### 3. 持續進化

改變了工作流？增加了新步驟？PML 會自動適應。

- 新的模式會被識別
- 你不再使用的舊模式會逐漸淡化
- 你的能力始終與你實際的工作方式保持同步

## 使用體驗是什麼樣的

**沒有 PML 時：**
> "我要部署。讓我想想……先跑測試，然後建構，再推送到映像檔倉庫，接著更新服務，最後通知團隊……"

**有了 PML：**
> "部署這個。"
>
> PML：「收到，正在執行你常用的部署流程。」

認知負擔消失了。你只需關注**想做什麼**，而不是**怎麼做**。

## 能力的組合

真正強大的地方在於：能力可以互相組合。

假設你已經有了：
- 「執行測試」能力
- 「建構 Docker」能力
- 「部署到雲端」能力

連續使用它們幾次後，PML 會自動建立：
- 「完整發佈」元能力

你的工作流可以自由組合，複雜性被逐層吸收，你在更高的抽象層級上工作。

PML 不會要求你預先定義工作流。

它觀察、它學習、它建議。

你用得越多，它越了解你。它越了解你，你就越少做重複的事。

**你的 AI，終於記住了你的工作方式。**
